컬렉션 프레임워크란 프로그램을 개발할때 사용하는 자료를 관리하는 자료구조

자료구조는 프로그램 실행 중 메모리에 자료를 유지,관리하기 위해 사용

이러한 자료구조를 자바가 미리 구현하여 java.util패키지에 제공하고있는데

이걸 컬렉션 프레임 워크라고함


Collection 인터페이스 - 하나의 자료를 모아서 관리

List 인터페이스 : ArrayList, Vector, LinkedList 클래스가 있음
1.순서있는 자료관리 , 중복 허용
2.0부터 인덱스를 사용하여 요소에 접근
추가 : list.add(값); // 앞에서부터 순서대로 추가
        list.add(추가할 위치 index, 값);	// 지정한 인덱스에 값을 추가, 그 뒤 값은 밀려난다.
1.for문
2.향상된 for문
3.반복자(Iterator)사용
	반복자는 java.util 패키지에 정의되어있는 Iterator 인터페이스를 구현하는 객체
	컬렉션의 원소들에 접근하는것이 목적
사용메서드
hasNext() : 아직 접근하지 않은 요소가 있으면 true를 반환
next() : 다음 요소를 반환 ... Object 타입을 반환 => Object : 가장 큰 클래스 자료형
remove()	: 최근에 반환된 요소를 삭제한다.

while문을 이용해 출력


 
Set 인터페이스 : HashSet, TreeSet 클래스가 있음
1.순서없는 자료관리, 중복허용불가

팩토리 메서드(factory method)
객체를 생성하는 정적 팩토리 메서드이다.
이것은 디자인 패턴의 하나로서 객체를 만드는 부분을 자식클래스에게 위임하는 패턴
즉 new 연산자를 호출하여 객체를 생성하는 코드를 자식클래스에게 위임하는 패턴
사용하는 이유는 하나의 클래스가 변경되었을 경우에 다른 클래스의 변경을 최소화

팩토리 메서드 이름으로 많이 사용되는건은 getInstance() 이다
인터페이스에서 팩토리 메서드를 제공할 때 정적 메서드를 사용하게 된다
static이므로 클래스명 .정적메서드로 접근
